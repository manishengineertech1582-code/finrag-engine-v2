# Plan 2 — Fix BM25 Dict Crash + Reranker Bypass

## Root Causes

### RCA-4 (critical): BM25 receives `dict` instead of `str`
- `_SafeEmbeddingsFilter` is a plain Python class, not a `BaseRetriever`
- `create_retrieval_chain` detects non-BaseRetriever and passes full `{"input": "..."}` dict
- `__getattr__` on `_SafeEmbeddingsFilter` delegates `invoke(dict)` to EnsembleRetriever
- EnsembleRetriever passes dict to BM25 → `dict.split()` → AttributeError crash

### RCA-5 (silent): Reranker completely bypassed
- MultiQueryRetriever calls `self.retriever.invoke(query_string)` on `_SafeEmbeddingsFilter`
- `_SafeEmbeddingsFilter` has no `invoke` → `__getattr__` → `base_retriever.invoke`
- EnsembleRetriever is called directly, skipping all filtering/reranking logic
- Reranker safety net and threshold never execute

## Fix

### Change 1: Convert `_SafeEmbeddingsFilter` → proper `BaseRetriever`
- Inherit from `BaseRetriever`
- Add Pydantic fields with `ConfigDict(arbitrary_types_allowed=True)`
- Rename `get_relevant_documents` → `_get_relevant_documents`
- Remove `__getattr__` (no longer needed)
- `invoke` now routes through `_get_relevant_documents` correctly

### Change 2: Add `SafeQueryWrapper(BaseRetriever)` as outermost layer
- Normalizes dict/non-string inputs at system boundary
- `create_retrieval_chain` sees a BaseRetriever → extracts string before calling
- Defense-in-depth: handles future LangChain API changes
- Wrap final retriever in `get_retriever()` before return

## Files Changed
- `src/retriever.py` — structural refactor
- `tests/test_retrieval_pipeline.py` — new regression tests
