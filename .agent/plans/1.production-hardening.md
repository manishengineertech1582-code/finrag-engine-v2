# Plan 1: Production Hardening — FinRAG Engine v2

## Root Cause Summary (from audit)
- C1: Blocking I/O in async handlers (load/chunk/embed all synchronous)
- C2: FAISS race condition (no write locking)
- C3: Cache key bug (top_k missing from key)
- C4: Filename collision (no tenant namespace)
- C6: File size check AFTER full read (memory DoS)
- C7: Docker multi-worker split-brain (in-process cache)
- C8: pyproject.toml Python 3.14 (impossible version)
- H1: Settings class never imported (all code uses raw os.getenv)
- H2: No retry/timeout on OpenAI calls
- H4: Health check only checks directory existence, not index validity
- H5: No deduplication (same file re-indexed repeatedly)

## RAG Upgrades
- Hybrid search: BM25Retriever + FAISS/MMR via EnsembleRetriever (weights 0.3/0.7)
- Re-ranking: EmbeddingsFilter (ContextualCompressionRetriever, threshold=0.75)
- Tenacity retry: exponential backoff on OpenAI 429/5xx
- MultiQueryRetriever: LLM reused via module-level singleton
- Format-aware chunking: CSV=1500/0, PDF=1200/200, TXT=1000/150, DOCX=800/150
