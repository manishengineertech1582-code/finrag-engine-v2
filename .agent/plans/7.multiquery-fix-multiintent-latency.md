## Why Prompt-Based Approach

LangChain 0.3.7 `MultiQueryRetriever` (Pydantic v2):
- `model_config = {'extra': 'ignore', ...}` — unknown fields silently dropped at init
- Post-construction assignment: Pydantic v2 raises `ValueError` for unknown attributes
- `from_llm()` signature: no `num_queries` parameter

The prompt approach:
- Works across all LangChain 0.3.x versions
- No runtime attribute errors
- The prompt text is the documented way to influence generation count
- Clean, explicit, testable

---

## Multi-Intent Heuristic

Conservative by design — avoids false positives on valid single-intent questions:

| Input | Result |
|-------|--------|
| "What is revenue?" | False |
| "What is revenue? What are risks?" | True (two `?`) |
| "Explain revenue and also describe risks" | True ("and also") |
| "What are the risks and opportunities?" | False (single intent with "and") |

Heuristic can be extended in `_is_multi_intent_query` without touching the API contract.
