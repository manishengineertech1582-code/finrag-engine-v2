# 13. structured-patient-confidence-and-clause-budget.md

## Goal
Fix the remaining structured-patient defects visible in the latest runtime logs for `C:\Users\manis\Downloads\Production-Project\finrag-engine-old`.

## Observed Defects
1. Exact structured patient lookups return very low confidence (for example 0.24) because confidence still uses the generic `sources / top_k` heuristic.
2. Structured compound clauses ignore the reduced clause `top_k` budget and can emit too many rows/sources, which increases latency and token load.
3. Structured clause logs are noisy because clause pre-routing emits the same route log used for top-level single-query routing.

## Fix Strategy
1. Add structured confidence overrides for deterministic patient-table matches.
2. Respect the per-clause `top_k` budget when structured lookup is used inside compound queries.
3. Make structured routing logs context-aware so single-query and compound-clause paths are distinguishable.
4. Add regressions for confidence correctness and structured clause caps.

## Validation
- `tests\test_multiquery_multiintent.py`
- `tests\test_api.py`
- query hardening slice covering latency and response schema


## Outcome
- Structured single-query answers now use deterministic confidence overrides instead of the generic sources/top-k heuristic.
- Structured compound clauses now honor the per-clause result budget via max_results, which trims excess rows and source cards.
- Compound pre-routing no longer emits the top-level structured routing log, so observability cleanly distinguishes single-query vs compound-clause paths.
