# Plan 5 — UI Redesign + Retrieval Hardening

## Objective

Fix retrieval correctness regression (MultiQuery silently broken) and redesign the
query interface to production/investor quality: Perplexity/Glean feel with visible
evidence, wider layout, and accurate pipeline transparency.

---

## Root Cause Map

| ID | Area | Problem | Fix |
|----|------|---------|-----|
| RCA-7 | `src/retriever.py` | `MultiQueryRetriever.from_llm()` rejects `num_queries` as unexpected kwarg (LangChain 0.3); silent except block falls back to base retriever — MultiQuery never runs | Remove from `from_llm()` call; set `retriever.num_queries = n` as attribute post-construction |
| RCA-8 | `src/retriever.py` | Deprecated `get_relevant_documents` warnings | Already replaced with `.invoke()` in Phase 12; fixed completely by RCA-7 (MultiQuery now runs through proper `.invoke()` path) |
| RCA-10 | `frontend/` | Query UI cramped (max-w-3xl), sources hidden behind toggle, question has weak hierarchy, loading skeleton shows no question, developer-tool feel | Full redesign: AnswerDisplay + QueryPage |

---

## Files Changed

### Backend

- `src/retriever.py` — `_apply_multi_query`: removed `num_queries` from `from_llm()`;
  set `retriever.num_queries = num_queries` after construction

### Tests

- `tests/test_latency_hardening.py` — Updated `test_multi_query_num_queries_set_as_attribute`:
  asserts `num_queries` NOT passed to `from_llm()`, asserts it IS set as attribute on the
  returned mock retriever

### Frontend

- `frontend/src/components/query/AnswerDisplay.tsx` — Full redesign:
  - Question: left-accent border (violet), `text-[15px] font-medium`, replaces tiny Q-bubble
  - Answer: `text-[15px] leading-[1.75]` (was `text-sm`), full-width card
  - Sources: **visible by default** — top 3 `SourceCard`s inline, "Show N more" toggle
  - Footer: `ConfidenceMeter` + `Zap` pipeline tags (always visible) + latency clock + details toggle
  - Retrieval detail panel: expandable, includes request_id slice
  - Removed unused `animationDone` state side-effect

- `frontend/src/features/rag/QueryPage.tsx` — Layout + state hardening:
  - `max-w-3xl` → `max-w-4xl` (wider conversation + composer)
  - `pendingQuestion` state captures question text before query dispatches;
    shown in loading skeleton so user sees what is being processed
  - Loading skeleton moved inside `rounded-xl border` card (matches answer card shape)
  - Stage messages updated: 900 ms → stage 1, 2600 ms → stage 2
  - `space-y-8` → `space-y-10` between conversation turns (more breathing room)
  - "Index is empty" warning shown above composer when noIndex + has prior history
  - Import cleanup: removed unused `MessageSquare`, added `Loader2`, `Search`

## Risk Notes

- `retriever.num_queries` is a standard LangChain `MultiQueryRetriever` instance attribute
  (confirmed in source). Setting it post-construction is the documented pattern for runtime config.
- Sources now shown by default: this increases DOM size for queries with many sources.
  `SOURCES_INLINE = 3` constant controls the visible count; easy to adjust.
- `pendingQuestion` state is reset to `null` when `isPending` clears — no risk of stale
  display if query is cancelled.
