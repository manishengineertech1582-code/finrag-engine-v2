# Plan 3 — UI Redesign + Retrieval Transparency

## Objective

Transform the UI from a dev tool into a production SaaS experience.
Eliminate all mismatches between ingestion, UI visibility, and retrieval clarity.

## Root Cause Summary

| Area | Problem | Fix |
|------|---------|-----|
| UI/Backend contract | `retrieval_meta` absent — UI had no data to show retrieval process | Added `RetrievalMeta` model to response |
| SourceCard | No content preview — sources were opaque filenames only | Added `snippet` field (first 200 chars of chunk) |
| Dashboard | Document count used session-only Zustand state — wiped on refresh | Now reads from persistent `GET /api/documents` |
| Loading state | Generic "Searching knowledge base…" message | "Analyzing documents… ⚡ Running hybrid retrieval + reranking" |
| AnswerDisplay | No retrieval transparency — black box AI answer | Added ⚡ badge: "Hybrid Search • MultiQuery • Reranked" + expandable counts |

## Files Changed

### Backend
- `models/response.py` — Added `RetrievalMeta` model; added `snippet: Optional[str]` to `ChunkSource`; added `retrieval_meta: Optional[RetrievalMeta]` to `QueryResponse`
- `app/routes/query.py` — Populated `snippet` from `doc.page_content[:200]`; populated `retrieval_meta` from settings + computed values

### Frontend
- `frontend/src/lib/api.ts` — Added `snippet` to `ChunkSource`; added `RetrievalMeta` interface; updated `QueryResponse`
- `frontend/src/state/appStore.ts` — Added `retrieval_meta?: RetrievalMeta | null` to `RecentQuery`
- `frontend/src/lib/hooks/useQuery.ts` — Passes `retrieval_meta` through to `addRecentQuery`
- `frontend/src/features/rag/QueryPage.tsx` — Passes `retrieval_meta` to `AnswerDisplay`; improved loading message
- `frontend/src/components/query/AnswerDisplay.tsx` — Added retrieval transparency badge + expandable detail row (queries_generated, candidates_before_rerank, candidates_after_rerank)
- `frontend/src/components/results/SourceCard.tsx` — Added snippet preview (collapsed: 2-line clamp; expanded: full content + metadata grid)
- `frontend/src/pages/Dashboard.tsx` — Switched document/chunk counts to `useDocuments` hook (persistent); "Recent uploads" → "Recent documents" from backend registry

### Tests
- `tests/test_response_schema.py` — 8 new schema regression tests: ChunkSource.snippet, RetrievalMeta defaults, QueryResponse.retrieval_meta, JSON round-trip
